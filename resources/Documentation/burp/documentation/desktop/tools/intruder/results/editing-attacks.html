Editing attacks

You can control the attack and modify the attack configuration while it is running. Do this from the results window.

Controlling an attack

With the results window open, you can control the attack from the top-level Attack menu. This enables you to carry out the following actions:

  • Pause the attack.
  • Resume the attack.
  • Restart the attack.

Related pages

  • For information on how to save and close an attack, see Saving attacks.

Configuring an attack

The results window includes clones of the original configuration tabs. You can use these to review and modify most aspects of the attack configuration while the attack is in progress.

You cannot modify aspects of the attack configuration that are fundamental to the structure of the attack. This includes:

  • The attack type.
  • The payload positions.
  • The payload type.

If you want to change these values, edit them in the original attack tab, then start a new attack.

Note

We recommend that you pause attacks before you modify your configuration. Changing the configuration while the attack is running may have unanticipated impacts, as changes will execute as each key is pressed.

For example, if you are using the Numbers payload type and delete a digit from the To field, the attack may abruptly complete because the To field now contains a smaller number.
